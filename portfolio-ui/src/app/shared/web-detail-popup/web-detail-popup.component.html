Java

L'application côté Serveur tourne sur les outils du langage de programmation Orienté Objet Java.

La version utilisée est Java 11 avec le JDK d'Amazone Correto.

Lien vers le Github du serveur

Spring Framework

Toujours côté serveur, l'application est construite avec le Framework Spring.

Spring Core pour l'utilisation du contexte applicatif Spring, Spring MVC pour la partie API, Spring Batch pour l'utilisation de batch dans les versions prochaines de l'application. Spring Data pour la liaison avec la Base de données (ici MongoDB).

Angular

Pour la partie Front, l'application est développée avec le Framework Angular.

La version 10 du Framework de Google est utilisé, elle consomme les API du Serveur Java notamment pour la partie envoie de mail. L'application est divisée en different module : - Core : pour la partie principale des différents chapitres de la page - Shared : pour le bandeau, le menu, le footer et les différantes pop-up - Service : pour les services de l'application (Appels API, Observable ...) - Model : pour les Class Model de l'application

Lien vers le Github de la partie Front

Docker

Docker est utilisé pour la conteneurisation de l'application.

Nous aurons trois conteneurs, un pour le serveur Java, un pour la base de données MongoDB et un pour la Front Angular. Docker Compose est utilisé pour créer le lien entre les trois conteneurs et pour l'orchestration de ceux-ci.

Les images Docker sont construites et hébergées sur le Github des différentes parties de l'application avec GitHub Actions. L'application comprenant le serveur, le front et la base de données peuvent être hébergés sur le même serveur physique ou virtule.
